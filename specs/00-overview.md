# BOSS FITNESS — обзор проекта

## Что это

Мобильное веб-приложение для премиального фитнес-клуба BOSS FITNESS: чек-ин по QR, выбор шкафчика, запись к тренерам, абонементы, история посещений. Источник дизайна — макет в корне репозитория (`tokens/`, `components/`, `ui_kits/mobile_app/`, `readme.md`).

**Сейчас в разработке:** только фронтенд + мок-бекенд. Реальная бизнес-логика (БД, авторизация, оплата, реальное распознавание QR) — следующий этап.

## Целевая платформа

Только телефоны. Адаптив под десктоп не делаем — вёрстка рассчитана на мобильный вьюпорт (`width=device-width`), без media query под планшеты/десктоп.

## Стек

| Слой | Технология |
|---|---|
| Frontend | Next.js (App Router, TypeScript), CSS Modules |
| Package manager | pnpm |
| Backend | FastAPI (Python), мок-данные в памяти |
| Контейнеризация | Docker + docker-compose |
| VCS | Git |

## Принципы

- **SOLID** — каждый компонент/роутер/функция отвечает за одну вещь (SRP); UI-слой не знает деталей HTTP (зависит от абстракции `lib/api/*`, а не от `fetch` напрямую) — это даёт возможность подменить мок на реальный бекенд без изменения экранов.
- **DRY** — дизайн-токены, типы и API-контракт описаны один раз и переиспользуются (токены → CSS-переменные → используются всеми компонентами; контракт API описан в `03-api.md` и зеркалится в Pydantic-схемах и TS-типах).
- **KISS** — никаких лишних абстракций на этом этапе: нет codegen, нет Redux/React Query, нет авторизации — простые fetch-хуки и серверные компоненты там, где это естественно.

## Spec-driven development

Перед написанием кода каждого слоя — соответствующий файл в `specs/`:
1. `01-design-tokens.md` — токены дизайна
2. `02-screens/*.md` — спека каждого экрана (маршрут, данные, состояния, переходы)
3. `03-api.md` — контракт мок-бекенда
4. `04-components.md` — контракты UI-компонентов
5. `05-nextjs-primer.md` — шпаргалка по Next.js (т.к. это первый проект на Next.js)

Код пишется по этим спекам, а не наоборот.

## Быстрый старт

Без Docker:

```bash
# backend
cd backend
python -m venv .venv && .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# frontend (другой терминал)
cd frontend
pnpm install
pnpm dev
```

Открыть `http://localhost:3000` в режиме мобильного вьюпорта DevTools.

С Docker:

```bash
docker compose up --build
```

> Примечание: корневой `readme.md` — это документация исходной дизайн-системы/макета (не трогаем). Этот файл (`specs/00-overview.md`) — точка входа в документацию самого приложения.
